# Rendering

Lawtext provides the functionality of rendering [`StdEL`](/lib/core/modules/core_src_law_std_stdEL.md#stdel) tree into several formats such as Lawtext, [Standard Law XML](https://elaws.e-gov.go.jp/file/XMLSchemaForJapaneseLaw_v3.xsd), HTML, and docx.

## Rendering into Lawtext

The rendering process into Lawtext is basically the reverse transformation of [parsing](/technical/parsing). However, the Lawtext renderer skips the creation of [`VirtualLine`](/lib/core/modules/core_src_parser_std_virtualLine.md#virtualline)s and directly transforms the [`StdEL`](/lib/core/modules/core_src_law_std_stdEL.md#stdel) tree into [`Line`](/lib/core/modules/core_src_node_cst_line.md#line)s.

You can find the entry point of the Lawtext renderer at [`renderLawtext()`](/lib/core/modules/core_src_renderer_lawtext/#renderlawtext).

## Rendering into Standard Law XML

The rendering process into Standard Law XML is just serializing the [`StdEL`](/lib/core/modules/core_src_law_std_stdEL.md#stdel) tree and adding an XML declaration on top of that.

You can find the entry point of the Standard Law XML renderer at [`renderXML()`](/lib/core/modules/core_src_renderer_xml/#renderxml).

## Rendering into HTML

The rendering process into HTML utilizes [React DOM Server API](https://react.dev/reference/react-dom/server). The HTML renderer first constructs a tree of React components from the [`StdEL`](/lib/core/modules/core_src_law_std_stdEL.md#stdel) tree and then converts the component into HTML string using [`renderToStaticMarkup()`](https://react.dev/reference/react-dom/server/renderToStaticMarkup#rendertostaticmarkup).

The same React component such as [`HTMLLaw`](/lib/core/modules/core_src_renderer_rules_law/#htmllaw), can be utilized to create React web apps that render `StdEL`.

[`renderHTMLfragment()`](/lib/core/modules/core_src_renderer_html/#renderhtmlfragment) renders the HTML fragment directly from `renderToStaticMarkup()`, and [`renderHTML()`](/lib/core/modules/core_src_renderer_html/#renderhtml) adds some other parts such as an HTML declaration and styles to make a stand-alone HTML file.

## Rendering into docx

Like the HTML renderer, the rendering process into docx utilizes [React DOM Server API](https://react.dev/reference/react-dom/server) but builds a WordprocessingML of [Office Open XML](https://www.ecma-international.org/publications-and-standards/standards/ecma-376/).

[`renderDocxAsync()`](/lib/core/modules/core_src_renderer_docx/#renderdocxasync) builds a docx binary as Office Open XML.
